再看InvokerTransformer类：

构造函数接受三个参数并赋值到this.iMethodName、this.iParamTypes、this.iArgs变量中。之后调transform方法会先获取参数input的类，然后通过反射的方式获取设定好的方法(this.iMethodName)并传递参数(this.iArgs)调用该方法。

也就是说InvokerTransformer类的构造函数设定好要调用的方法和传递的参数，然后调用此类的transform方法的时，传进去的对象都会被传递设定好的函数参数，然后调用设定好的函数。

再看回chainedTransformer的transform方法循环调用，执行流程是这样子的：

  • 取出new ConstantTransformer(Runtime.class)对象，调用其transform方法返回Runtime.class类对象
  • 取出new InvokerTransformer("getMethod", new Class[]{String.class, Class[].class}, new Object[]{"getRuntime", new Class[0]})对象，调用transform方法并且把上一步transform返回的Runtime.class类对象作为参数。根据我们刚才分析的InvokerTransformer的transform方法的代码逻辑，可以知道Runtime.class的getMethod方法会被调用，并且传递的参数是"getRuntime"，即Runtime.class.getMethod("getRuntime")，并返回Runtime对象。
  • 取出new InvokerTransformer("exec", new Class[]{String.class}, new Object[]{"calc"})对象，同理调用上面一步返回的Runtime对象的exec方法，传递参数为"calc"。

上面整个流程实现了Runtime.getRuntime().exec("calc")这一整个调用。
